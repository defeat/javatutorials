Creating a class

In this section, you will learn about classes, the building blocks of Java

A class can be:

  • Public
  • Private
  • Protected
  • An internet server

In the headline of the class:

  • You need the keyword “static”
  • You need the name of the class
  • You need to write "public"
  • You need the type of the class

The name of the Java file:

  • Needs to be the same as the class it contains, with the .class extension
  • Needs to be the same as the class it contains, with the .java extension
  • Needs both the .class and .java extensions
  • Needs no extension

Static and Dynamic

In this section, you will learn the difference between static and dynamic variables

The static context is created:

  • It is not created, ever
  • At compile time
  • When the file is created
  • At runtime

The dynamic context is created:

  • At runtime
  • At compile time
  • It is not created, ever
  • When the file is created

A non-static variable:

  • Need the word “static”
  • Is allocated memory for at compile time
  • Cannot be referenced from a static context
  • Does not exist in Java

Return Statements

Here you will learn the concept of returning statements in Java.

The keyword “void” means:

  • The method is neither declared in a static context nor in a dynamic one
  • The method does not return a value
  • The return type is of type “void”
  • The method cannot be accessed from within the class

You can use a method that returns a value:

  • By passing it to a variable of the same type as the method
  • By simply calling it on a new line
  • By printing it on the standard output
  • By writing it in the class heading

Compile & Run

In this section, you will learn how to compile your java classes with javac and running them on the terminal

Javac is the name of:

  • Any Java program
  • The Java virtual machine
  • The Java interpreter
  • The Java compiler

The command “java” is used for:

  • The Java compiler
  • The Java Interpeter
  • The Java Documentation book
  • Java help

System.out.println is used for:

  • Printing a primitive type
  • Printing a piece of paper
  • Printing a picture
  • Printing the content of the .class file

Method Arguments

Here is where you will learn how to call different methods to do various functions in your code.

Method Arguments...

  • Need to be the same type as the method they are declared in
  • Need to be primitive types
  • Can be of type String
  • Can be called by Javac

You can have

  • Negative number of Method Arguments
  • Zero or more Method Arguments
  • Voice recording as Method Argument
  • Pictures as Method Arguments

When you call a function with Method Arguments..

  • You don’t need to care about the type required
  • You can provide more arguments than required
  • You can provide less arguments than required
  • You need to enclose the arguments passed in parenthesis
